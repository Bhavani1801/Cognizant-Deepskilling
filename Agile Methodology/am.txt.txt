

# ?? Module 13 – Agile Methodology 
---

## ?? Overview

This module provides a comprehensive understanding of **Agile methodology**, focusing on its:

* Core principles and values
* Agile Manifesto
* Scrum framework
* Estimation and planning techniques
* User story creation

Agile promotes:

* **Iterative development**
* **Collaboration**
* **Continuous delivery of value**

It contrasts with traditional models like **Waterfall**, emphasizing flexibility and adaptability.

---

## ?? Learning Objectives

After completing this module, you will be able to:

* Explain Agile principles, values, and the Agile Manifesto
* Differentiate between **Agile vs Waterfall models**
* Understand Scrum roles, ceremonies, and artifacts
* Apply estimation techniques (Story Points, Planning Poker)
* Interpret velocity and burndown charts
* Understand sprint planning and iterative delivery
* Write effective user stories using **INVEST principle**
* Define acceptance criteria using **Given-When-Then format**
* Collaborate effectively in Agile teams

---

## ?? Key Concepts Explained

---

### ?? 1. Agile Methodology

Agile is a software development approach based on:

* Iterative progress
* Customer collaboration
* Continuous feedback

---

### ?? 2. Agile Manifesto

**4 Core Values:**

* Individuals & interactions over processes & tools
* Working software over documentation
* Customer collaboration over contract negotiation
* Responding to change over following a plan

**12 Principles:**
Focus on customer satisfaction, frequent delivery, simplicity, and adaptability.

---

### ?? 3. Agile vs Waterfall

| Agile               | Waterfall        |
| ------------------- | ---------------- |
| Iterative           | Sequential       |
| Flexible            | Rigid            |
| Continuous feedback | Minimal feedback |
| Fast delivery       | Late delivery    |

---

### ?? 4. Scrum Framework

#### ? Roles

* **Scrum Master** ? Facilitator
* **Product Owner** ? Defines requirements
* **Development Team** ? Builds product

---

#### ? Ceremonies

* Sprint Planning
* Daily Scrum
* Sprint Review
* Sprint Retrospective

---

#### ? Artifacts

* Product Backlog
* Sprint Backlog
* Increment
* Definition of Done

---

### ?? 5. Agile Estimation & Planning

#### ? Story Points

* Measure effort (not time)
* Based on complexity

---

#### ? Planning Poker

* Team-based estimation technique
* Uses cards (e.g., Fibonacci sequence)

---

#### ? Velocity

* Work completed per sprint

---

#### ? Burndown Chart

* Shows remaining work vs time

---

### ?? 6. Agile User Stories

#### ? Format:

```
As a [user], I want [goal], so that [benefit]
```

---

#### ? INVEST Principle:

* **I**ndependent
* **N**egotiable
* **V**aluable
* **E**stimable
* **S**mall
* **T**estable

---

#### ? Acceptance Criteria

Defined using:

```
Given [condition]
When [action]
Then [result]
```

## ?? Best Practices

* Encourage team collaboration

## ?? Summary

Agile methodology enables teams to:

* Adapt quickly
* Deliver continuously
* Improve collaboration
* Build high-quality software

## ?? Learning Objectives

After completing this module, you will be able to:

  * Few-shot
  * Chain-of-thought
* Follow best practices for AI usage
* Understand GitHub Copilot features
* Set up Copilot in development environment
* Generate code, documentation, and test cases

---

## ?? Key Concepts Explained

---

### ?? 1. Generative AI

Generative AI refers to AI systems that can:

* Generate text
* Produce code
* Create images or content

Examples:

* Chatbots
* Code assistants
* Content generators

---

### ?? 2. Traditional AI vs Generative AI

| Traditional AI          | Generative AI        |
| ----------------------- | -------------------- |
| Rule-based / predictive | Content creation     |
| Classification tasks    | Text/code generation |
| Fixed outputs           | Dynamic outputs      |

---

### ?? 3. Prompt Engineering

Designing effective inputs to get desired AI outputs.

#### ? Techniques:

* **Zero-shot** ? No examples
* **Few-shot** ? Provide examples
* **Chain-of-thought** ? Step-by-step reasoning

---
