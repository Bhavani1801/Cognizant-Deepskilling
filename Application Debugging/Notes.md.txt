* Identifying frontend issues
* Inspecting the DOM structure
* Debugging JavaScript and TypeScript code
* Monitoring application state

The module emphasizes **systematic debugging techniques** to enhance:

* Code quality
* Application performance
* User experience

## ??? Tools & Technologies

### ?? Chrome DevTools

A powerful browser-based debugging tool used for:

* DOM inspection
* JavaScript debugging
* Performance monitoring

**Key Features:**

* Elements panel (DOM tree)
* Sources panel (code debugging)
* Console (logs & errors)
* Network panel (API calls)

---

### ?? Visual Studio Code (VS Code)

A lightweight yet powerful code editor used for advanced debugging.

**Capabilities:**

* Integrated debugger
* Breakpoint management
* Launch configurations (`launch.json`)
* Extensions for Angular debugging

## ?? Key Concepts Explained

---

### ?? 1. Debugging with Chrome DevTools

Chrome DevTools allows developers to debug applications directly in the browser.

**Process:**

1. Run Angular app:

   ```bash
   ng serve
   ```
2. Open browser ? Right click ? Inspect
3. Navigate to **Sources tab**

**Debugging Features:**

* Set breakpoints
* Step through execution
* Inspect call stack
* Monitor variables

---

### ?? 2. DOM Inspection

DOM (Document Object Model) represents the structure of a webpage.

**Use Cases:**

* Identify UI bugs
* Fix layout issues
* Modify elements in real time

**Example:**

* Change CSS properties live
* Check element hierarchy

---

### ?? 3. JavaScript & TypeScript Debugging

Angular uses TypeScript, which is compiled to JavaScript.

**Problem:**

* Browser executes JS, not TS

**Solution: Source Maps**

* Map compiled JS back to TS
* Enable debugging in original TS files

**Benefits:**

* Easier debugging
* Better readability
* Accurate error tracing

---

### ?? 4. Breakpoints & Call Stack

#### ? Breakpoints

Pause execution at a specific line of code.

**Types:**

* Line breakpoints
* Conditional breakpoints
* Event listener breakpoints

---

#### ? Call Stack

Shows the sequence of function calls leading to the current execution point.

**Helps in:**

* Understanding execution flow
* Identifying error origins

---

### ?? 5. Debugging with VS Code

VS Code enables advanced debugging outside the browser.

**Steps:**

1. Install extensions (Angular, Debugger for Chrome)
2. Create `launch.json` configuration
3. Start debugging session

**Features:**

* Attach to running Angular app
* Step over / step into code
* Watch variables
* Debug TypeScript directly

---
