# ?? MODULE 11 – DEVOPS AND CI/CD (DETAILED NOTES)

---

## ?? Overview

DevOps is a **modern software development methodology** that integrates:

* Development (Dev)
* Operations (Ops)

It focuses on **automation, collaboration, continuous monitoring, and faster delivery** of applications.

Traditionally:

* Developers write code
* Operations deploy & maintain

?? DevOps removes this separation and creates a **continuous workflow**.

---

## ?? Learning Objectives

After completing this module, learners will be able to:

* Understand DevOps concepts and lifecycle
* Explain Continuous Integration (CI) and Continuous Delivery (CD)
* Identify popular DevOps tools and platforms

---

## ?? 1. Introduction to DevOps

### ?? What is DevOps?

DevOps is a combination of:

* **CULTURE** ? collaboration between teams
* **PRACTICES** ? automation & monitoring
* **TOOLS** ? CI/CD pipelines

---

### ?? Goals of DevOps

* Faster software delivery
* Improved collaboration
* Continuous feedback
* High-quality applications

---

### ? Benefits of DevOps

* ?? Faster releases
* ?? Continuous improvement
* ?? Early bug detection
* ?? Reduced downtime
* ?? Better team coordination

---

### ?? DevOps Lifecycle

1. **Planning** ? Requirement gathering
2. **Development** ? Writing code
3. **Build** ? Compile & package
4. **Testing** ? Automated testing
5. **Release** ? Prepare deployment
6. **Deployment** ? Deploy to production
7. **Operations** ? Maintain system
8. **Monitoring** ? Track performance

?? This cycle is continuous and iterative.

---

## ?? 2. Continuous Integration (CI)

### ?? Definition

CI is the practice of **automatically integrating code changes** into a shared repository frequently.

---

### ?? How CI Works

1. Developer commits code
2. Code pushed to repository (Git)
3. CI server triggers build
4. Automated tests run
5. Feedback generated

---

### ?? Advantages

* Detect bugs early
* Reduce integration problems
* Improve code quality

---

## ?? 3. Continuous Delivery (CD)

### ?? Definition

Continuous Delivery ensures that:

* Code is always ready for deployment
* Deployment can be done anytime

---

### ?? Workflow

* CI builds ? tested ? packaged ? ready for release

---

## ?? 4. Continuous Deployment

### ?? Definition

* Fully automated deployment
* No manual approval required

?? Every change that passes testing is deployed automatically.

---

## ?? CI vs CD vs Continuous Deployment

| Feature     | CI           | Continuous Delivery | Continuous Deployment |
| ----------- | ------------ | ------------------- | --------------------- |
| Purpose     | Integration  | Ready to deploy     | Auto deployment       |
| Automation  | Build & test | Release prep        | Full pipeline         |
| Manual Step | No           | Yes (deploy)        | No                    |

## ??? 4. Basic Docker Commands

```
docker pull <image>        # Download image
docker run <image>         # Run container
docker ps                  # List running containers
docker images              # List images
docker stop <id>           # Stop container
docker rm <id>             # Remove container
docker build -t name .     # Build image
```
